Roadmap

API and WebApp

The initial prototype is built with Typescript, Express, Mongo and Angular. I've chosen Express for the initial prototype since the intention was to create an easy to pick up open source tool. However, in the future, when ModernSoftware/WSF is more ready for production deployments. I will be creating a poll to vote for continuing work in Express or WSF.

Ethereum Custodial and Non-Custodial Wallets

The initial prototype features both Custodial and Non-Custodial Wallet solutions for Ethereum. This is inherently much easier to implement for ETH since Metamask exists. The reason I've omitted BTC and XMR from the initial release is because there is no good solution for BTC or XMR that is non-custodial. See this post by Abra and this post by Evercoin to learn more about the difference between custodial and non-custodial wallets. The intention is for DecentralizedCreator to be friendly to beginner users who may only know how to deal with a custodial solution. But, also catering to more advanced users who may prefer hardware wallets or, to simply just own their private keys. Our intention is to *always* encourage users to use non-custodial solutions however.

One Click Deploys on %eaxops

Despite %eaxops.com not being in public beta (or ready for production scale orchestration). I do intend on creating one click deploys for DecentralizedCreator with %eaxops. This will make it a lot easier to setup your own custom version of DecentralizedCreator, if you do intend on publishing your own version of DecentralizedCreator.

Bitcoin and Monero Custodial and Non-Custodial Wallets

Currently there is no real open source solution for Non-Custodial wallets for Bitcoin and Monero. Since non-custodial wallet APIs for Bitcoin and Monero full nodes would be extremely valuable. It would make sense to separate this aspect of the project to two projects. Non-Custodial wallets are becoming increasingly more important to the integrity of the Cryptocurrency Community in order to protect individual's assets. See the potential Quadriga exit scam as to why having your wallets managed by a third party are a liability if said party were to be compromised. Unfortunately, solutions such as BTCPay don't function this way, and, essentially create a centralized database of wallets which is what DecentralizedCreator wants to avoid and eventually deprecate in the future.

IPFS Search Engine for Video, Audio and Text

DecentralizedCreator's second phase will be a fully fledged search engine for Video, Audio and Text content. Think of it as a replacement for platforms such as YouTube. Content can be posted unencrypted, or, require a PGP Key in order to access it. PGP Keys can be managed in a non-custodial fashion, or, managed via our custodial database. Currently, I am working on an optimized IPFS node for serving content via HTTP called ModernSoftware/SFS which is always open to contributions.
